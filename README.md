# Suppression-Aware County-Level Epilepsy and Status Epilepticus Mortality in the United States, 2019-2024

Code, processed aggregate data, tables, figures, and documentation for a suppression-aware county-level analysis of epilepsy/status epilepticus mortality in the United States, 2019-2024.

## Repository contents

- `src/`: reproducible Python analysis scripts.
- `config/`: analysis configuration, if present.
- `data/raw/wonder/`: frozen aggregate CDC WONDER query outputs copied from the validated extraction project.
- `data/raw/covariates/`: public county covariate inputs used in the analysis.
- `data/raw/geography/`: county geography files used for mapping.
- `data/processed/`: derived analytic datasets used for modeling and figures.
- `tables/`: analysis and manuscript-ready tables.
- `figures/`: maps and plots generated by the analysis.
- `reports/`: analysis, audit, and provenance reports.
- `docs/`: data dictionary and reproducibility notes.

## Data sources

Primary mortality data were obtained from CDC WONDER Multiple Cause of Death, 2018-2024, Single Race. County covariates include CDC/ATSDR Social Vulnerability Index, USDA Rural-Urban Continuum Codes, NCHS urban-rural classification, American Community Survey county variables, and county geography files.

No person-level data are included in this repository. The repository contains aggregate public mortality outputs, processed aggregate datasets, code, tables, figures, and documentation.

## Suppression-aware analysis frame

CDC WONDER suppresses small county death counts. This project preserves exact, suppressed, and explicit zero county cells as distinct statuses. Suppressed county cells are represented as bounded 1-9 death intervals and are not treated as zero. The analysis evaluates rurality-associated mortality patterns using observed-only models, bias-bounding scenarios, residual-allocation analyses, and interval-likelihood negative binomial models.

## Basic reproducibility

Install dependencies:

```bash
pip install -r requirements.txt
```

Run the full analysis from the repository root:

```bash
python src/run_all.py
```

The repository is intended to reproduce the processed aggregate-data analysis from the staged inputs. It does not automate CDC WONDER browser extraction.
